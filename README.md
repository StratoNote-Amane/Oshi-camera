# 推しカメラ (prototype)

カメラ映像に推し(MMDモデル)を重ねて、その場で配置調整してから撮影するプロトタイプです。
ビルド不要の静的サイト（HTML/CSS/JS + Three.js CDN）なので、GitHub Pagesにそのまま置けます。

## 動作環境

- iPhone 14 Safari（iOS Safari）を想定
- HTTPS環境が必須（`getUserMedia` はHTTPでは動作しません）→ GitHub Pagesは標準でHTTPSなのでOK
- Flowlyとは別リポジトリ・別URLで運用する想定

## セットアップ手順（GitHub側）

1. GitHubで新規リポジトリを作成する（例: `oshi-camera`）。Publicにしておくと無料でPagesが使えます。
2. このフォルダの中身一式（`index.html` / `style.css` / `main.js` / `assets/` / `README.md`）をそのままリポジトリのルートに置いてpushする。
3. リポジトリの **Settings → Pages** を開き、Source を「Deploy from a branch」、Branch を `main` / `/(root)` に設定して保存する。
4. 数分後、`https://(あなたのユーザー名).github.io/oshi-camera/` でアクセスできるようになります。
5. iPhoneのSafariでそのURLを開き、共有ボタン→「ホーム画面に追加」しておくとアプリのように起動できます。

## 使い方（アプリ内操作）

- 起動画面で「カメラを起動する」をタップ → カメラ許可を求められたら許可する
- 1本指でドラッグ：推しを上下左右に移動
- 2本指でピンチ：拡大縮小
- 2本指でひねる：向き（回転）を調整
- 右上の ⟲ ボタン：位置を初期状態にリセット
- 右上の ⟳ ボタン：前面/背面カメラの切り替え
- 下の白いボタン：撮影 → 画像プレビューが出るので**長押しして「写真に保存」**

## 今回のスコープ（あえて実装していないこと）

ご相談の通り、まず「動くこと」を優先したMVPです。以下は次のステップとして意図的に見送っています。

- スカート・髪の物理演算（揺れもの）→ 静止ポーズのみ
- ARKitのような自動平面検出・光源推定・オクルージョン → 手動配置のみ（iPhone 14はLiDAR非搭載のため元々オクルージョンは不可）
- 複数キャラクターの選択・切り替えUI → 現状は天音かなたモデル1体のみ固定読み込み
- ポーズ/モーション切り替え → 現状はPMXのデフォルト（Tポーズに近い）姿勢のまま
- MMDらしい輪郭線（アウトライン表現）→ 未実装（three.jsのOutlineEffect等で後日追加可能）

## 既知の調整ポイント

- **モデルの大きさ**: MMDモデルは「1ユニット≒8cm」という慣習を仮定して `main.js` 内の `MMD_UNIT_TO_METER = 0.081` を設定していますが、実機で見て大きすぎる/小さすぎる場合はこの数値を直接調整するか、アプリ内でピンチ操作して都度合わせてください。
- **カメラのFOV(画角)**: `main.js` の `camera` のFOVは45度で仮置きしています。iPhone実機のレンズ画角とズレていると、拡縮・移動の感覚が「実際のカメラで動かした量」と若干ズレることがあります。実機で試して違和感があれば、この数値を調整してください。
- **フロントカメラ利用時**: プレビューは鏡写し表示にしていますが、推しモデル自体は反転していません（実装をシンプルにするため）。気になる場合は教えてください、対応します。

## テクスチャの命名について

PMX内部の参照名の一部（`tex_kanata_face parts.png` / `tex_kanata_short pants.png`、いずれも半角スペース区切り）が、いただいたファイル名（アンダースコア区切り）と異なっていたため、`assets/kanata/tex_kanata/` にはPMXが期待する名前（スペース区切り）でコピーしてあります。今後テクスチャを差し替える際は、この命名ルールに注意してください。
