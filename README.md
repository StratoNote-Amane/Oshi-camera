# 推しカメラ (prototype)

カメラ映像に推し(MMDモデル)を重ねて、その場で配置調整してから撮影するプロトタイプです。
ビルド不要の静的サイト（HTML/CSS/JS + Three.js CDN）なので、GitHub Pagesにそのまま置けます。

## 動作環境

- iPhone 14 Safari（iOS Safari）を想定
- HTTPS環境が必須（`getUserMedia` はHTTPでは動作しません）→ GitHub Pagesは標準でHTTPSなのでOK
- Flowlyとは別リポジトリ・別URLで運用する想定

## セットアップ手順（GitHub側）

1. GitHubで新規リポジトリを作成する（例: `oshi-camera`）。Publicにしておくと無料でPagesが使えます。
2. このフォルダの中身一式（`index.html` / `style.css` / `main.js` / `assets/` / `README.md`）をそのままリポジトリのルートに置いてpushする。
3. リポジトリの **Settings → Pages** を開き、Source を「Deploy from a branch」、Branch を `main` / `/(root)` に設定して保存する。
4. 数分後、`https://(あなたのユーザー名).github.io/oshi-camera/` でアクセスできるようになります。
5. iPhoneのSafariでそのURLを開き、共有ボタン→「ホーム画面に追加」しておくとアプリのように起動できます。

## 使い方（アプリ内操作）

- （2体目以降を追加した場合のみ）最初にキャラクター選択画面が出るので、一緒に撮影する推しを選ぶ
- 準備画面で「カメラを起動する」をタップ → カメラ許可・向き（ジャイロ）許可を求められたら両方許可する
  - ジャイロを許可すると、スマホを動かしても推しがその場に立っているように見える（後述）
  - 許可しなくても撮影自体は可能（その場合は画面固定の従来モードで動作）
- 1本指でドラッグ：推しを上下左右に移動
- 2本指でピンチ：拡大縮小
- 2本指でひねる：向き（回転）を調整
- 画面下の顔アイコン：表情を切り替え（通常/笑顔/ウインク/びっくり/困り）
- 右上の ⟲ ボタン：位置と「その場に固定する基準の向き」を両方リセット（移動して撮影場所を変えた時はこれを押す）
- 右上の ⟳ ボタン：前面/背面カメラの切り替え
- 下の白いボタン：撮影 → プレビューが出るので「保存・共有する」ボタン（iOSの共有シートから直接「写真に保存」できる）か、画像を長押しして保存

## 今回追加した機能

- **ジャイロAR（3DoF）**: `DeviceOrientationEvent` でスマホの向きを取得し、基準姿勢からの差分回転をthree.jsのカメラに適用。位置(左右前後)まではARKitのような自動追従はできないが、向き（見回す動き）には推しがその場に立っているように追従する。⟲ボタンで「今の向き」を新しい基準として再固定できる。
- **表情切り替え**: PMXに内蔓の表情モーフ（頂点モーフ）を解析し、天音かなたモデルに実在する26個のモーフから「笑顔」「ウインク」「びっくり」「困り」のプリセットを作成。ボタンで選ぶと滑らかに(補間して)切り替わる。
- **自動まばたき**: 2〜5秒間隔でランダムに自動まばたきするアニメーションを追加（生きている感じを出すための演出、表情プリセットとは独立して動作）。
- **MMDらしい輪郭線**: three.js公式の`OutlineEffect`を使用。MMDLoaderがPMXの材質情報からエッジ色・太さを自動的に読み込むため、追加の素材や設定なしでMMDらしい黒い輪郭線が出る。
- **二重接地影**: 足元にタイトで濃い接地影＋広がる柔らかい影の2層構成にして、地面についている感じを強化。
- **キャラクター選択の枠組み**: `main.js`の`CHARACTERS`配列にエントリを追加するだけで選択画面が自動的に複数カード表示に切り替わる設計（現在は1体のため選択画面は自動的にスキップされる）。

## 既知の制限（ジャイロAR）

- 位置（左右・前後・上下の移動）の自動追従はできない。ジャイロでできるのは「向き（回転）」のみ。歩いて移動すると推しの立ち位置はズレるので、その場合は⟲ボタンで再配置・再固定してください。
- ジャイロの基準は「起動した瞬間」or「⟲を押した瞬間」のスマホの向き。方位磁石のように絶対的な向きを使っているわけではないので、その場でしばらく使う分には問題ないが、長時間経つと若干ズレることがある。
- 1本指ドラッグでの移動量計算はカメラが正面（回転なし）を向いている前提の簡易計算のため、スマホを大きく傾けた状態でのドラッグ操作は感覚が多少ズレることがある。

## 今回のスコープ（あえて実装していないこと）

- スカート・髪の物理演算（揺れもの）→ 静止ポーズのみ
- ARKitのような自動平面検出・位置の自動追従・オクルージョン → 手動配置＋向きのみジャイロ追従（iPhone 14はLiDAR非搭載のため元々オクルージョンは不可）
- 複数キャラクターの同時配置（ツーショット）→ 現状は1体を選んで撮影する方式のみ
- カメラで自分の表情を読み取ってリアルタイムにミラーする機能 → 現状はボタンでの固定表情切り替えのみ
- 周囲の明るさに合わせたモデルの陰影・色合いの自動調整 → 優先度が低いとのことで見送り（次のステップ候補）


## 既知の調整ポイント

- **モデルの大きさ**: MMDモデルは「1ユニット≒8cm」という慣習を仮定して `main.js` 内の `MMD_UNIT_TO_METER = 0.081` を設定していますが、実機で見て大きすぎる/小さすぎる場合はこの数値を直接調整するか、アプリ内でピンチ操作して都度合わせてください。
- **カメラのFOV(画角)**: `main.js` の `camera` のFOVは45度で仮置きしています。iPhone実機のレンズ画角とズレていると、拡縮・移動の感覚が「実際のカメラで動かした量」と若干ズレることがあります。実機で試して違和感があれば、この数値を調整してください。
- **フロントカメラ利用時**: プレビューは鏡写し表示にしていますが、推しモデル自体は反転していません（実装をシンプルにするため）。気になる場合は教えてください、対応します。

## テクスチャの命名について

PMX内部の参照名の一部（`tex_kanata_face parts.png` / `tex_kanata_short pants.png`、いずれも半角スペース区切り）が、いただいたファイル名（アンダースコア区切り）と異なっていたため、`assets/kanata/tex_kanata/` にはPMXが期待する名前（スペース区切り）でコピーしてあります。今後テクスチャを差し替える際は、この命名ルールに注意してください。

## 今回追加した機能(2)：ポーズテンプレート／脱MMD調整

### ポーズテンプレート
「直立／ピース／手を振る／腰に手」の4種類をボーン回転のプリセットとして実装しました。ボーンはbind-pose(初期姿勢)からの相対回転として定義しているので、モデル固有の初期姿勢がどうであっても崩れにくい設計です。

**重要**: 角度の数値は実機で描画を確認せずに作った「たたき台」です。恐らく最初は変な方向に曲がったりします。そのための**ポーズ調整モード**です。

#### ポーズ調整モードの使い方
1. 右上の🛠ボタンをタップ
2. 下に出てくるパネルで、動かしたいボーンを選択
3. X/Y/Zのスライダーで角度をその場で調整（即座に反映されます）
4. 良い角度になったら「JSONをコピー」→ コピーされた内容をこちらに貼って送ってください。`main.js`の該当ポーズの`bones`の値を差し替えます
5. 「このポーズを初期値に戻す」でいつでもたたき台の数値に戻せます

### MMDらしさを抑える調整（アニメ調で写真に馴染ませる）
- **輪郭線を細く・薄く**: `OutlineEffect`のパラメータに加え、モデル読み込み後に各材質の`outlineParameters`を上書きして太さを約4割に、不透明度も下げた
- **トゥーンの陰影を滑らかに**: MMDの陰影は通常「くっきり2色に分かれる」段差のあるグラデーション画像(ドット絵的なNearest補間)を使うが、これを`LinearFilter`に変更して段差を滑らかにした
- **リムライト追加**: 背景に馴染ませるため、後ろ・横から柔らかい青みがかった縁光を追加（実写合成でよく使われる手法）

これらは実機で見て「まだMMD感が強い」ようであれば、追加でトゥーン自体をやめて通常のPBRシェーディングに切り替える等、さらに踏み込んだ調整も可能です。見た感想を聞かせてください。
