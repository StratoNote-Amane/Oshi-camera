# CONSTRAINTS.md — 推しカメラ (oshi-camera)

AI開発憲法.md の定めに従い、本プロジェクト固有の制約をここに定義する。
本憲法を書き換えず、個別事情はすべてこのファイルで管理する。

## 1. スコープの境界

### 対象 (In-scope)
- iPhone (iOS Safari) 向けに、カメラ映像へMMDモデルを重ねてその場で配置調整し撮影するWebアプリ
- 1体のキャラクターを選んで一緒に撮影する体験（切り替え式）
- 表情プリセット切り替え（PMX内蔵の表情モーフを使用）
- ポーズプリセット切り替え（ボーン回転、bind-pose相対）
- 向き（回転）のみのジャイロAR固定（3DoF、センサーノイズ低減のダンピング付き）
- 手ぶれ対策（動きが収まるまで待ってから撮影、真のOIS/EISではなくソフトウェア的近似）
- セルフタイマー・構図グリッド・撮影時のハプティクス/フラッシュ演出等のUI仕上げ
- 将来的な2D透過素材キャラクターへの対応（アーキテクチャとして準備済み）

### 対象外 (Out-of-scope、明示的に見送り)
- スカート・髪の物理演算（揺れもの）
- 位置（平行移動）の自動AR追従・平面検出・オクルージョン（ARKit相当機能）
- 複数キャラクターの同時配置（ツーショット撮影）
- カメラで自分の表情を読み取るリアルタイムミラーリング
- 脚のIKによる重心移動ポーズ（片足重心など、座りポーズはボーン直接回転による近似で代替済み）
- サーバー/バックエンド、ユーザーアカウント、課金、外部への配布・公開
- AIによる環境認識（人物・物体検出、深度推定、光源方向推定）
  → 現時点では対象外だが、将来の実装方針は`docs/VISION_REALISM.md`に合意済み。
    ブラウザ内完結(ONNX Runtime Web等)・ノービルド・サーバーレスの方針を維持できる
    ことを条件に、段階的な対象内への格上げを検討する。

## 2. 環境・コスト制約

- **開発環境**: Windows PCのみ。Mac/Xcodeは使用不可 → ネイティブAR (Unity+ARKit) は不採用
- **実行環境**: iPhone 14 Safari（LiDAR非搭載 → オクルージョンは元々不可）
- **ホスティング**: GitHub Pages（無料の静的ホスティング）。Flowlyとは別リポジトリ・別URL
- **ビルド**: ビルドステップなし（npm/webpack等の導入なし）。Three.jsはCDN配信のESモジュールを使用
- **コスト**: 無料枠のみで完結させる（有料API・サーバー不使用）
- **利用範囲**: 個人利用のみ。公開・販売は行わない（天音かなたのファンメイドMMDモデルを
  使用しているため、商用配布はライセンス上の懸念がある）。「商業アプリ並みの品質・使い心地」
  を目指す場合も、公開はせず個人利用の範囲内でのUI/UX・安定性の作り込みに留める。

## 3. 技術の禁止/推奨リスト

- **禁止**: Unity、ARKit/ARCoreの直接利用、サーバーサイド処理、ビルドツール必須の構成
- **禁止（理由=Windows開発+ノービルド維持のため）**: npm経由のバンドルを前提とするライブラリ導入
- **推奨**: three.js（CDN配信のESモジュール）、素のHTML/CSS/JS

## 4. リポジトリ構成の例外運用

AI開発憲法は「`/docs`はすべての仕様書を格納し実装コードの混入を禁止する」と定めるが、
本プロジェクトはGitHub Pagesでの無ビルド配信という制約上、実装ファイル
(`index.html` / `main.js` / `style.css` / `assets/`) はリポジトリ直下に置く。
`/docs`は本憲法が定める通り仕様書専用のまま維持する。

## 5. 精度・品質に関する現実的な制約

- PMXバイナリ解析だけでは、ボーンの正確な回転軸・可動域までは分からない。
  ポーズ・表情の数値は「たたき台」であり、実機での調整（アプリ内ポーズ調整モード）を
  前提としたワークフローで運用する（詳細はDECISION_LOG.md ADR-005）。
- Claude（開発者役）はブラウザ実機での見た目を直接確認できないため、
  視覚的な最終判断（ポーズの角度・輪郭線の太さ等）は必ずユーザーの実機確認を経る。
- 見た目に関する不具合(色味・明るさ等)は、憶測で直さず、可能な範囲で使用ライブラリ
  (three.js等)の実ソースコードを取得・確認してから対応する（詳細はDECISION_LOG.md ADR-006）。
  このサンドボックス環境ではヘッドレスブラウザによる実レンダリング確認はできない
  （Chromiumがsnap配布のみでsnapdがストアに接続できないため）。
