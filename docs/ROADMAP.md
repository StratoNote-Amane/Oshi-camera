# ROADMAP.md — 開発計画・タスクバックログ

AI開発憲法における「タスク仕様書」に相当するが、個人開発の規模に合わせて
1ファイルに集約している（1タスク1ファイルにする程の規模ではないため）。
タスク完了時、仕様や設計に関わる決定があればDECISION_LOG.mdにADRを追記する。

## 進め方の基本方針

Claudeは実機のレンダリング結果を直接確認できない。そのため、見た目や操作感に
関わるタスクは「Claudeが実装 → ユーザーが実機(iPhone 14 Safari)で確認 →
フィードバック → Claudeが修正」のループを前提に進める。憶測で完璧を狙わず、
まず動くものを素早く出して実機で検証するサイクルを優先する。

## 運用ルール（このプロジェクトでの開発サイクル）

1. Claude側で実装 → zipで納品
2. ユーザーが実機(iPhone 14 Safari)で動作確認
3. 不具合・違和感をフィードバック（ポーズ関連はポーズ調整モードのJSON出力を活用すると早い）
4. Claude側で修正。設計判断を伴う場合はDECISION_LOG.mdにADRを追記
5. スコープ自体を拡げる・変える大きな変更をする時は、着手前にCONSTRAINTS.mdを更新する
