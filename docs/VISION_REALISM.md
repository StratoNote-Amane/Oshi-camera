# VISION_REALISM.md — 「写真とCGを一つの映像へ融合させる」指針

ユーザーからの開発記録(2026年頃)を元にした、今後の改善指針。ROADMAP.mdの元ネタとして
このファイルに集約し、優先順位や個々の改善案の詳細はここを参照する。

## 設計思想

> 「MMDをリアルへ近付ける」のではなく、**「写真とCGを同じ映像表現へ統一する」**。
> リアルとアニメの中間を目指すのではなく、一枚の写真として違和感のない映像を作ることを
> 最終目標とする。個々のエフェクトを増やすことより、光・色・影・空気感・動きを
> 統一する設計を優先する。

## AIによる環境認識（将来構想）

現時点では実装しないが、方向性として合意している将来構想。

```
Camera → AI Environment Analyzer(シーン認識/人物検出/物体検出/深度推定/光源推定)
       → Scene State → Three.js Renderer
```

- AIはレンダリングを担当せず、周囲の環境を解析してScene Stateに反映する役割のみ
- 技術候補: ONNX Runtime Web / TensorFlow.js / MediaPipe（すべてブラウザ内実行、
  GitHub Pages・ノービルド・サーバーレスの現行方針を維持できるもの）
- 想定用途: 人物・物体の検出（椅子/テーブル/壁など）→ 座る/隣に立つ等の自動配置、
  深度推定 → 物陰へ隠れる簡易オクルージョン・接地感向上、光源推定 → ライティング自動反映
- CONSTRAINTS.mdの「対象外」に現状は含まれるが、実装コストと効果を見ながら
  段階的に対象内へ格上げを検討する

## Claude側の制約について（重要）

このプロジェクトはClaudeが実機のレンダリング結果を直接確認できない前提で進めている
（CONSTRAINTS.md 5節）。今回追加した項目のうち、特に以下は「実装はしたが目視確認なし」
の状態であることに注意:

- ポーズ(座る/考え中/直立の非対称)の関節角度
- 微細アニメーションの回転軸(髪の揺れが不自然な方向を向いている可能性がある)
- 環境光推定の反応の強さ(明るさの変化に対して大げさすぎる/弱すぎる可能性がある)

実機確認の上、違和感があれば数値を教えてもらえれば調整する。
